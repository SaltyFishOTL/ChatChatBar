# 朋友圈 MCP 版设计 v2

## Summary

做最小可用朋友圈：根 Tab 时间线 + AI 自动生成动态 + 用户点赞。朋友圈只作前端展示与沉浸氛围，不进入聊天主线，不写长期记忆。

## Core Rules

- 全局设置：`开启朋友圈功能`，默认关闭；关闭时隐藏根 Tab 且不生成。
- 角色设置：`开启朋友圈`，默认关闭。
- AI 只判断当前时机是否适合发。
- 活跃门槛：48 小时内有交流。
- 推进门槛：若已有上一条朋友圈，AI 先只根据长期记忆、上一条朋友圈判断是否有足够新推进。
- 频率：单卡每 2-13 小时预排候选时间。
- 限额：单卡每天最多 4 条；全局每天最多 18 条。
- 补发：不限制条数；按已排到期时间逐条补发。
- 仅用户可见：基础赞数固定 0；用户点赞后显示本地点赞 +1；显示小锁/“仅你可见”。

## Generation Flow

- 分层生成：先只发长期记忆、上一条朋友圈让 AI 判断该不该发；该发时再发角色卡（不含开场白、备用开场白）、长期记忆、最近 3 条完整聊天消息、上一条朋友圈生成动态。
- 朋友圈 AI、NovelAI Prompt 设计 AI 使用当前默认对话模型及其原始参数，不覆盖 thinking budget、max tokens 等模型参数。
- 生成 AI 输出：发送者、文案、图片内容设计、可见性、赞数 tier、生成理由。
- 文案强调私密感：像角色不经意发出的碎片、暗示、只给少数人看的情绪，不复述聊天记录。
- 图片设计强调照片感、私密感、生活切片；必要时使用偷拍视角、低角度、门缝/镜面/远处抓拍/手机随手拍等构图。
- 图片内容设计交给 NovelAI 提示词设计 AI，转成 NAI tag prompt。
- NovelAI 生成照片风格图片；失败则保留待重试/失败状态。
- 文案 0-60 字，短、含蓄、像真人朋友圈。

## UX

- 根 Tab：`聊天 / 朋友圈 / 社区 / 管理`，仅全局开启后显示。
- 时间线对标微信/QQ：白底、头像、昵称、文案、单图、时间、点赞按钮、点赞数。
- 无评论输入框，无评论列表。
- 点赞按钮可切换；公开动态显示总赞数；仅用户可见动态基础赞数为 0，用户点赞后显示 1。
- 开启全局功能后检查后台可靠性：自启动、通知、电池优化、后台限制；受限则引导设置。
- 管理页新增朋友圈调试区：可指定角色卡立即生成新朋友圈，并查看判定、生成、NovelAI Prompt 设计、NovelAI 生图的完整输入输出。调试入口用于排错，可记录 AI 判定但不被排程、限额、48 小时门槛阻断。
- 底部 Tab 红点：存在未查看的新朋友圈时，朋友圈按钮右上角显示红点；打开朋友圈根页后清除。存在未查看的新助手聊天消息时，聊天按钮右上角显示红点；打开聊天根列表后清除。

## Test Scenarios

- 全局关闭时 Tab 隐藏，后台不生成。
- 角色未开启时不发朋友圈。
- 48 小时无交流时不发。
- 无足够推进时不发。
- 排程、单卡/全局限额、补发规则生效。
- 点赞/取消点赞只影响本地展示。
- 不存在评论、转聊天、长期记忆入口。
- 调试入口可指定角色卡立即生成；页面显示 AI 全量输入输出，成功结果会保存为朋友圈动态。
- 新朋友圈未查看时底部朋友圈按钮显示红点，进入朋友圈后清除。
- 新助手消息未查看时底部聊天按钮显示红点，进入聊天根列表后清除；用户自己发送的新消息不触发聊天红点。
